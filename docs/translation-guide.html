데일리바이블 역본 만들기 가이드

데일리바이블 앱은 .dbbible.json 파일을 가져와 새 역본이나 주석성경으로 읽을 수 있습니다. 이 문서는 그 파일의 공식 포맷 명세입니다. 사람이 직접 만들어도 되고, 이 페이지 주소를 AI에게 주고 만들어 달라고 해도 됩니다.

저작권 안내 (반드시 읽어주세요)
본인이 적법하게 보유·이용할 수 있는 본문으로만 파일을 만들어 주세요. 저작권이 있는 역본(예: 개역개정, NIV 등)으로 만든 파일을 다른 사람에게 배포·공유하는 것은 저작권 침해입니다. 주석성경도 베이스 본문의 저작권을 그대로 따르므로, 공유는 공개도메인 본문(예: 개역한글)으로 만든 것만 하세요. 이 사이트는 저작권이 있는 역본 파일을 제공하지 않습니다.

AI에게 만들어 달라고 하기

ChatGPT·Claude 등 파일을 만들 수 있는 AI에게 아래처럼 요청하세요.

다음 가이드 페이지를 읽고 포맷을 정확히 지켜서
데일리바이블 가져오기 파일(.dbbible.json)을 만들어 줘:
https://dailybible.kr/docs/translation-guide.html

[여기에 본인이 보유한 본문/주석 자료를 붙여넣거나 첨부]

- 역본 이름: (예: 개역한글+내주석)
- 역본 ID: (예: KRV-mynote)
- 페이지의 "기계 판독용 명세(JSON Schema)"와 "추가 규칙"을 기준으로 만들고,
  페이지의 "검증 스크립트(Python)"를 실행해 [오류] 0건을 확인한 뒤에만
  .dbbible.json 파일로 저장해 줘.

받은 파일은 앱의 설정 → 성경 가져오기에서 불러오면 됩니다. (안드로이드는 파일 관리자에서 파일을 탭해 "데일리바이블"로 여는 것도 가능)

JSON을 직접 다루기 어렵다면 역본 스튜디오에서 파일을 열어 화면으로 편집할 수도 있습니다 (열기 → 절·주석·메모 수정 → 내려받기).

파일 형식 요약

UTF-8 인코딩의 단일 JSON 파일이며, 최대 30MB입니다. 최상위 구조:

필드타입설명
formatstring"dailybible-translation@1" 또는 "dailybible-translation@2" (전체 문자열 — "@2"만 쓰면 안 됨). memo 필드나 제2경전 책이 하나라도 있으면 반드시 @2, 아니면 @1(구버전 호환) (필수, 고정값)
idstring역본 고유 ID. 영문/숫자/-/_ 1~32자. 같은 ID로 다시 가져오면 덮어씀
namestring앱에 표시될 이름. 자유 텍스트 (예: "개역한글+내주석")
languagestring언어 코드 ("ko", "en" 등). TTS 언어 선택에 사용
directionstring"ltr" 또는 "rtl"
booksarray포함된 책 목록. 66권 전체가 아니어도 됨(부분 정경 허용)
contentobject책 ID → 본문. books의 ID 집합과 정확히 일치해야 함

books 항목

{ "id": "GEN", "name": "창세기", "nameEn": "Genesis", "testament": "old", "chapters": 50 }
  • id: 아래 66권 표준 ID 중 하나 (필수)
  • testament: "old" 또는 "new"
  • chapters: 생략 가능 — 실제 본문에서 자동 계산됩니다

content 항목 (본문)

"GEN": {
  "bookId": "GEN",
  "bookName": "창세기",
  "chapters": [
    {
      "chapter": 1,
      "verses": [
        { "verse": null, "text": "천지 창조" },
        { "verse": 1, "text": "태초에 하나님이 천지를 창조하시니라",
          "note": "히브리어 '바라'(창조하다)는 오직 하나님을 주어로만 쓰인다." },
        { "verse": 2, "text": "땅이 혼돈하고 공허하며 …" }
      ]
    }
  ]
}
  • chapter: 1부터 연속이어야 합니다 (1, 2, 3, … 건너뛰기 불가)
  • verse: 절 번호(양의 정수). 특수값 — 0은 시편 표제, null은 번호 없는 소제목(섹션 제목) 행
  • text: 절 본문 (필수, 문자열)
  • note: 선택. 이 절에 대한 주석. 있으면 앱에서 절 아래 회색 글씨로 표시되고, 음성 낭독에서는 읽지 않습니다
  • memo: 선택(@2 전용). 직전 공유자의 메모 한 세대. 앱에서 초록 글씨로 표시되며, 이 파일을 기반으로 다시 공유하면 주석(note)으로 합쳐집니다(승격)

주석성경 만들기

공개도메인 본문(예: 개역한글)에 note를 붙이면 나만의 주석성경이 됩니다. 앱에서는 별도 역본으로 나타나 개역한글 ↔ 개역한글+내주석처럼 넘겨가며 읽을 수 있습니다.
  • name은 알아보기 쉽게: "개역한글+내주석", "개역한글+김목사주석"
  • id는 원본과 다르게: "KRV-mynote" (같은 ID는 덮어쓰므로, 주석 개정판 배포에 활용 가능)
  • 일부 책만 담아도 됩니다 — 로마서 주석만 담은 파일도 유효
  • 완성 파일은 앱의 내보내기 기능으로 다른 기기·다른 사람에게 전달할 수 있습니다 (공개도메인 본문일 때만!)
  • 재공유와 승격: 받은 파일(주석+메모)을 기반으로 앱의 "주석성경 만들기"로 새 버전을 만들면, 이전 공유자의 메모(memo)는 주석(note) 뒤에 줄바꿈으로 합쳐지고(승격), 내 메모가 새 memo가 됩니다. 즉 파일 안의 메모는 항상 "직전 공유자 한 사람"의 것입니다

66권 표준 책 ID

구약(39권):

GEN EXO LEV NUM DEU JOS JDG RUT 1SA 2SA 1KI 2KI 1CH 2CH EZR NEH EST JOB
PSA PRO ECC SNG ISA JER LAM EZK DAN HOS JOL AMO OBA JON MIC NAH HAB ZEP
HAG ZEC MAL

신약(27권):

MAT MRK LUK JHN ACT ROM 1CO 2CO GAL EPH PHP COL 1TH 2TH 1TI 2TI TIT PHM
HEB JAS 1PE 2PE 1JN 2JN 3JN JUD REV

가톨릭 정경 제2경전(@2 형식 전용) — 구약 영역에 배치되며, 표시 순서는 books 배열 순서를 따릅니다:

TOB JDT WIS SIR BAR 1MA 2MA
ESG LJE S3Y SUS BEL   (다니엘/에스델 추가부 — 본문 통합을 권장, 별도 책도 허용)

책 이름(name·bookName)은 교단 표기 그대로 자유롭게 쓰면 됩니다 — 가톨릭 역본이라면 "마태오 복음서", "탈출기", "요한 묵시록"처럼. ID는 내부 식별자일 뿐이고 화면에는 항상 이 이름이 표시됩니다.

완성 예시 (전체 파일)

{

  "id": "KRV-mynote",
  "name": "개역한글+내주석",
  "language": "ko",
  "direction": "ltr",
  "books": [
    { "id": "GEN", "name": "창세기", "nameEn": "Genesis", "testament": "old" }
  ],
  "content": {
    "GEN": {
      "bookId": "GEN",
      "bookName": "창세기",
      "chapters": [
        {
          "chapter": 1,
          "verses": [
            { "verse": 1, "text": "태초에 하나님이 천지를 창조하시니라",
              "note": "성경 전체의 서문. 하나님이 만물의 창조주이심을 선언한다." },
            { "verse": 2, "text": "땅이 혼돈하고 공허하며 흑암이 깊음 위에 있고 하나님의 신은 수면에 운행하시니라" }
          ]
        }

    }
  }
}

기계 판독용 명세 (AI용 JSON Schema)

AI/프로그램용: 파일을 생성하거나 검사할 때는 아래 스키마와 추가 규칙을 기준으로 삼으세요. (위 본문 설명과 동일한 내용의 형식 명세입니다 — 앱 검증기와 동기화됨, 2026-07-07 기준)

{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "dailybible .dbbible.json",
  "type": "object",
  "required": ["format", "id", "name", "books", "content"],
  "properties": {
    "format": { "enum": ["dailybible-translation@1", "dailybible-translation@2"] },
    "id": { "type": "string", "pattern": "^[A-Za-z0-9_-]{1,32}$" },
    "name": { "type": "string", "minLength": 1 },
    "language": { "type": "string" },
    "direction": { "enum": ["ltr", "rtl"] },
    "books": {
      "type": "array", "minItems": 1,
      "items": {
        "type": "object", "required": ["id"],
        "properties": {
          "id": { "type": "string" },
          "name": { "type": "string" },
          "nameEn": { "type": "string" },
          "testament": { "enum": ["old", "new"] },
          "chapters": { "type": "integer", "minimum": 1 }
        }
      }
    },
    "content": {
      "type": "object", "minProperties": 1,
      "additionalProperties": {
        "type": "object", "required": ["bookId", "chapters"],
        "properties": {
          "bookId": { "type": "string" },
          "bookName": { "type": "string" },
          "chapters": {
            "type": "array", "minItems": 1,
            "items": {
              "type": "object", "required": ["chapter", "verses"],
              "properties": {
                "chapter": { "type": "integer", "minimum": 1 },
                "verses": {
                  "type": "array", "minItems": 1,
                  "items": {
                    "type": "object", "required": ["verse", "text"],
                    "properties": {
                      "verse": { "type": ["integer", "null"], "minimum": 0 },
                      "text": { "type": "string" },
                      "note": { "type": "string" },
                      "memo": { "type": "string" }
                    }
                  }
                }
              }
            }
          }
        }
      }
    }
  }
}

스키마로 표현되지 않는 추가 규칙 — 모두 지켜야 앱이 파일을 받습니다:

  1. content의 키 집합은 books[].id 집합과 정확히 일치 (양방향)
  2. content["X"].bookId는 키 "X"와 동일
  3. 각 책의 chapter는 1부터 연속 (1, 2, 3, … 건너뛰기·중복 불가)
  4. 책 ID는 66권 표준 목록(위 참조)에 있어야 함 — 제2경전 ID는 @2 형식에서만 허용
  5. memo 필드가 하나라도 있거나 제2경전 책이 있으면 format은 반드시 "dailybible-translation@2"
  6. 파일 전체는 UTF-8, 30MB 이하 (초과 시 책을 나눠 여러 파일로)
  7. verse 특수값: 0 = 시편 표제, null = 번호 없는 소제목 행

검증 스크립트 (Python)

완성한 파일이 올바른지 아래 스크립트로 확인할 수 있습니다 (표준 라이브러리만 사용, Python 3.8+). AI에게 파일을 만들게 했다면, 이 스크립트를 실행해 [오류] 0건을 확인한 결과와 함께 달라고 하세요.

python3 validate_dbbible.py 내파일.dbbible.json

[오류]는 앱이 가져오기를 거부하는 문제, [경고]는 동작하지만 고치면 좋은 문제입니다.

#!/usr/bin/env python3
# 데일리바이블 .dbbible.json 검증 스크립트 (포맷 @1/@2 기준, 앱 검증기와 동일 규칙)
# 사용법: python3 validate_dbbible.py 파일.dbbible.json
# 결과: [오류] = 앱에서 가져오기 실패, [경고] = 동작하지만 권장 수정. 오류 0건이면 exit 0.
import json
import os
import re
import sys

CANON = set(
    "GEN EXO LEV NUM DEU JOS JDG RUT 1SA 2SA 1KI 2KI 1CH 2CH EZR NEH "
    "EST JOB PSA PRO ECC SNG ISA JER LAM EZK DAN HOS JOL AMO OBA JON "
    "MIC NAH HAB ZEP HAG ZEC MAL MAT MRK LUK JHN ACT ROM 1CO 2CO GAL "
    "EPH PHP COL 1TH 2TH 1TI 2TI TIT PHM HEB JAS 1PE 2PE 1JN 2JN 3JN "
    "JUD REV".split()
)
DEUTERO = set("TOB JDT WIS SIR BAR 1MA 2MA ESG LJE S3Y SUS BEL".split())
NT = set(
    "MAT MRK LUK JHN ACT ROM 1CO 2CO GAL EPH PHP COL 1TH 2TH 1TI 2TI "
    "TIT PHM HEB JAS 1PE 2PE 1JN 2JN 3JN JUD REV".split()
)
ID_RE = re.compile(r"^[A-Za-z0-9_-]{1,32}$")
MAX_BYTES = 30 * 1024 * 1024

errors = []
warnings = []


def err(msg):
    errors.append(msg)


def warn(msg):
    warnings.append(msg)


def is_int(v):
    return isinstance(v, int) and not isinstance(v, bool)


def finish(stats=None):
    for w in warnings:
        print("[경고]", w)
    for e in errors:
        print("[오류]", e)
    if errors:
        print("\n검증 실패 — 오류 %d건 (앱에서 가져오기가 거부됩니다)" % len(errors))
        sys.exit(1)
    if stats:
        print("검증 통과 ✓  책 %d권 · %d장 · %d절 (경고 %d건)" % (*stats, len(warnings)))
    sys.exit(0)


def check_verses(bid, ch_num, verses, state):
    if not isinstance(verses, list) or not verses:
        err("%s %d장: verses 배열이 비었습니다." % (bid, ch_num))
        return
    seen_nums = set()
    for v in verses:
        verse_ok = isinstance(v, dict) and (
            v.get("verse") is None or (is_int(v.get("verse")) and v["verse"] >= 0)
        )
        if not verse_ok or not isinstance(v.get("text"), str):
            err("%s %d장: 절 형식 오류 — verse는 null/0/양의 정수, text는 문자열이어야 합니다." % (bid, ch_num))
            return
        state["verses"] += 1
        if is_int(v.get("verse")):
            if v["verse"] in seen_nums:
                warn("%s %d장: 절 번호 중복(%d)" % (bid, ch_num, v["verse"]))
            seen_nums.add(v["verse"])
        if not v["text"].strip():
            warn("%s %d장 %s절: text가 비어 있습니다." % (bid, ch_num, v.get("verse")))
        if isinstance(v.get("memo"), str) and v["memo"].strip():
            state["needs_v2"] = True


def main(path):
    if not os.path.isfile(path):
        err("파일을 찾을 수 없습니다: %s" % path)
        finish()
    if os.path.getsize(path) > MAX_BYTES:
        err("파일이 30MB를 초과합니다 — 책을 나눠 여러 파일로 만드세요.")
    try:
        with open(path, encoding="utf-8", errors="strict") as f:
            raw = json.load(f)
    except UnicodeDecodeError:
        err("UTF-8 인코딩이 아닙니다.")
        finish()
    except json.JSONDecodeError as e:
        err("JSON 문법 오류: %s" % e)
        finish()
    if not isinstance(raw, dict):
        err("최상위가 JSON 객체가 아닙니다.")
        finish()

    fmt = raw.get("format")
    is_v2 = fmt == "dailybible-translation@2"
    if fmt not in ("dailybible-translation@1", "dailybible-translation@2"):
        err('format 값 오류: %r — 정확히 "dailybible-translation@1" 또는 "dailybible-translation@2"' % fmt)

    tid = raw.get("id")
    if not isinstance(tid, str) or not ID_RE.match(tid):
        err("id: 영문/숫자/-/_ 조합 1~32자여야 합니다.")
    if not isinstance(raw.get("name"), str) or not raw["name"].strip():
        err("name(역본 이름)이 비어 있습니다.")
    if not isinstance(raw.get("language"), str) or not raw.get("language"):
        warn("language가 없습니다 — TTS 언어 선택에 쓰이므로 'ko', 'en' 등을 권장합니다.")
    if raw.get("direction") not in ("ltr", "rtl"):
        warn("direction이 'ltr'/'rtl'이 아니어서 앱이 'ltr'로 처리합니다.")

    books = raw.get("books")
    content = raw.get("content")
    if not isinstance(books, list) or not books:
        err("books 배열이 비었습니다.")
        finish()
    if not isinstance(content, dict):
        err("content 객체가 없습니다.")
        finish()

    state = {"verses": 0, "needs_v2": False}
    seen = set()
    for i, b in enumerate(books):
        if not isinstance(b, dict) or not isinstance(b.get("id"), str):
            err("books[%d]: 객체 형식이 아니거나 id가 없습니다." % i)
            continue
        bid = b["id"]
        if bid in DEUTERO:
            state["needs_v2"] = True
        elif bid not in CANON:
            err("books[%d]: 알 수 없는 책 ID '%s' — 표준 ID(대문자, 예: GEN)를 쓰세요." % (i, bid))
        if bid in seen:
            err("중복된 책 ID: %s" % bid)
        seen.add(bid)
        t = b.get("testament")
        if t not in ("old", "new"):
            warn("%s: testament가 없거나 잘못돼 앱이 'old'로 처리합니다." % bid)
        elif bid in NT and t != "new":
            warn("%s: 신약 책인데 testament가 'new'가 아닙니다." % bid)
        elif (bid in CANON and bid not in NT or bid in DEUTERO) and t != "old":
            warn("%s: 구약 영역 책인데 testament가 'old'가 아닙니다." % bid)

    ckeys = set(content.keys())
    if ckeys != seen:
        only_books = sorted(seen - ckeys)
        only_content = sorted(ckeys - seen)
        if only_books:
            err("books에만 있고 content에 없는 책: %s" % ", ".join(only_books))
        if only_content:
            err("content에만 있고 books에 없는 책: %s" % ", ".join(only_content))

    n_chapters = 0
    for bid in sorted(ckeys & seen):
        book = content[bid]
        if not isinstance(book, dict):
            err("%s: content 값이 객체가 아닙니다." % bid)
            continue
        if book.get("bookId") != bid:
            err("%s: bookId(%r)가 content 키와 일치하지 않습니다." % (bid, book.get("bookId")))
        chapters = book.get("chapters")
        if not isinstance(chapters, list) or not chapters:
            err("%s: chapters 배열이 비었습니다." % bid)
            continue
        prev = 0
        for c in chapters:
            if not isinstance(c, dict) or not is_int(c.get("chapter")) or c["chapter"] < 1:
                err("%s: 장 번호가 양의 정수가 아닙니다." % bid)
                break
            if c["chapter"] != prev + 1:
                err("%s: 장 번호는 1부터 연속이어야 합니다 (%d장 다음이 %s장)." % (bid, prev, c["chapter"]))
                break
            prev = c["chapter"]
            n_chapters += 1
            check_verses(bid, prev, c.get("verses"), state)

    if state["needs_v2"] and fmt == "dailybible-translation@1":
        err('memo 필드나 제2경전 책이 있으면 format이 "dailybible-translation@2"여야 합니다.')
    if is_v2 and not state["needs_v2"]:
        warn("@2 전용 요소(memo·제2경전)가 없습니다 — 구버전 호환을 위해 @1을 권장합니다.")

    finish(stats=(len(seen), n_chapters, state["verses"]))


if __name__ == "__main__":
    if len(sys.argv) != 2:
        print("사용법: python3 validate_dbbible.py 파일.dbbible.json")
        sys.exit(2)
    main(sys.argv[1])

자주 하는 실수

  • format 값 오타 — 정확히 dailybible-translation@1 또는 dailybible-translation@2 (축약형 "@2" 불가)
  • books에는 있는데 content에 없는 책(또는 반대) — 두 목록은 정확히 일치해야 함
  • 장 번호 건너뜀 — 반드시 1부터 연속
  • 표준 외 책 ID (Gen, gen 불가 — 대문자 GEN)
  • 30MB 초과 — 책을 나눠 여러 파일로 만드세요
